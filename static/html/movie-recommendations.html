Collaborative Filtering System¶

While content-based system relie (as the name suggests) on the content of the movies, collaborative systems relie on human interactions with the movies. Collaborative-filtering systems can use user informations to compare the user to other users with similarity metrics. Recommendations then are made based on what people who are similar to me like. The system that was developed here, however, is item based. Items are compared based on user feedback (e.g. reviews) with similarity metrics. Then, the items that are most closely related to the items that I like are recommended.

Let's discuss in more detail what such a system entails. To build our system, we used a dataset of critic reviews on Rotten Tomatoes (from Kaggle: https://www.kaggle.com/datasets/talha002/rottentomatoes-400k-review). We proceeded in the following steps:

  1. Reviewer-Movie-Matrix: The columns are the reviewers and the rows the movies. Each entry tells us what score the reviewer gave the movie (if any). If the reviewer did not write a review of a specific movie, we set the value to 0. (Most entries are zero, we say the matrix is sparse.)
  2. Movie-Similarity-Matrix: The Reviewer-Movie-Matrix gives us a vector of scores for each movie. We compare these vectors pairwise (here we use cosine similarity, but other metrics are possible) and create a new matrix that consists of these pairwise comparisons. The entries of this matrix tell us how similar each movie is to the other movies.
  3. Recommendations: When the user provides their favorite movies, the system checks the Movie-Similarity-Matrix and finds the movies that are closest to them by adding the cosine similarity between that movie and each of the favorite movies.

Recommendation System -- Collaborative-Filtering¶

Data Preparation¶

To actually build a recommendation system, we create a Item-User-Matrix. For each reviewer and movie, the matrix contains the score that the reviewer gave in their review. We note here that most of the entries are empty for this matrix. There are several ways to handle this. We will try out several ways for this project. We start with the easiest way of just filling the NaN values with zeros. This procedure makes the underlying assumption that movies are not reviewed because they are not of interest to the reviewer.

Movie Recommendations¶

Now that we have the similarity matrix, we can create a function that takes as input a set of movies with ratings and returns recommendations. In particular, the function takes as input the matrix that gives us the similarity of the movies as well as a set of movies and ratings. With the ratings as weights, it sums up the similarity values to find the movies that are most similar to the input movies. This function is used on the website for the collaborative filtering approach.

In [47]:
def get_movie_recommendations(user_movies, item_similarity_matrix, user_ratings):
    # Create an empty DataFrame to store the recommendations
    recommendations = pd.DataFrame({'Movie': item_similarity_df.index,'Score': [0 for k in range(len(item_similarity_df.index))]})

    # Loop through each movie in the user's list
    for movie in user_movies:
        # Get the similarity scores between the current movie and all other movies
        similarity_scores = item_similarity_matrix[movie]
        
        # Multiply the similarity scores by the user's ratings
        weighted_scores = similarity_scores * user_ratings[movie]

        # Add the weighted scores to the recommendations DataFrame
        recommendations['Score'] += weighted_scores.values

    # Filter out movies the user has already watched
    recommendations = recommendations[~recommendations.Movie.isin(user_movies)]
    
    # Sort the recommendations by score in descending order
    recommendations = recommendations.sort_values(by='Score', ascending=False)[:5]


    return recommendations.Movie.tolist()

Future Work - Collaborative Filtering¶

As mentioned before, this simple collaborative-filtering algorithm makes a strong assumption on the movies that have not been reviewed by a reviewer, namely that the reviewer does not like the movie. That is not necessarily the case.

So, we future work should build on another approach: Matrix Factorization. The idea behind this approach is to predict missing reviewer rating based on the review data. In particular, we build a machine learning model that learns lower-dimensional representations of the users and movies in a so called embedding space. The embeddings summarize the properties of the movies and reviewers and these properties are used to predict the missing ratings (as the dot products of the two embeddings). The model then minimizes the difference between predicted and actual ratings during training. By doing so, it learns patterns and relationships in the data, enabling it to make predictions for user-movie pairs it hasn't seen during training.

Due to the sparsity of this dataset, Matrix Factorization recommendation systems built upon this data set are not able to capture the specific features of the reviewers particularly well. So, implementing this approach on a new dataset is crucial to obtain relevant and personal movie recommendations.
